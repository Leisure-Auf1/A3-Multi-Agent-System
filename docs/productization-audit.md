## 3. Installation Experience Audit

### 3.1 Developer: Install Veritas-Core (for contributing)

```bash
git clone https://github.com/Leisure-Auf1/Veritas-Core.git
cd Veritas-Core
python -m venv .venv
source .venv/bin/activate
pip install -e .
pip install -e ".[dev]"
make test
```

**Assessment:** Works but requires manual steps. No `make install` shortcut.

**Issues:**
1. README says "A3" — users will be confused
2. No `.venv` creation is automated
3. No `CONTRIBUTING.md` to explain the workflow

### 3.2 Developer: Install A3 (application developer)

```bash
git clone https://github.com/Leisure-Auf1/A3-Multi-Agent-System.git
cd A3-Multi-Agent-System
python -m venv .venv
source .venv/bin/activate
# Need Veritas-Core available — not on PyPI!
pip install git+https://github.com/Leisure-Auf1/Veritas-Core.git  # BROKEN (no tag)
pip install -e /path/to/Veritas-Core  # Manual
pip install -r requirements.txt
streamlit run app.py
```

**Critical Issue:** `veritas-core>=7.0.0` in requirements.txt cannot be resolved automatically because Veritas-Core is not on PyPI and has no git tag.

### 3.3 End User: Deploy A3

Currently, Render deployment works only if Veritas-Core is manually installed first. The `render.yaml` `buildCommand` runs `pip install -r requirements.txt` which will fail because `veritas-core>=7.0.0` is not on PyPI.

### 3.4 CLI User: Use Veritas CLI

```bash
pip install veritas-core
veritas run --task "analyze" --agent "evaluator"
```

**Assessment:** Would work IF published to PyPI. Currently only works with `pip install -e .`.

## 5. Dependency Resolution Strategy

The core issue: A3 requires `veritas-core>=7.0.0` but Veritas-Core is not on PyPI.

### Option A: Publish to PyPI (Recommended)

```
pip install veritas-core  # from PyPI
pip install a3-multi-agent-system  # also from PyPI
```

**Pros:** Standard, resolves automatically  
**Cons:** Requires PyPI account, project name registration

### Option B: Git-based dependency

In A3's `requirements.txt`:
```
veritas-core @ git+https://github.com/Leisure-Auf1/Veritas-Core.git@v7.0.0
```

**Pros:** No PyPI needed, works immediately  
**Cons:** Requires git tag `v7.0.0`, every install clones repo

### Option C: Monorepo-lite (submodule)

```
git submodule add https://github.com/Leisure-Auf1/Veritas-Core.git veritas-core
```

**Pros:** Version-locked, local development  
**Cons:** Complex setup, not standard for Python

**Recommendation:** Publish to PyPI (Option A) + git tag as fallback (Option B).
