## 4. Release Blockers

### P0 — Must Fix Before v1.0.0

| # | Blocker | Impact | Fix |
|:--|:--------|:-------|:----|
| 1 | **Linux PyInstaller spec missing** | Cannot build Linux `.tar.gz` | Create `A3-Agent-linux.spec` using forward-slash paths |
| 2 | **Version strings stale** | `build.bat` says v7.1.0, `.spec` needs update | Bump all version references to v1.0.0 |
| 3 | **Hidden imports outdated** | `web.theme`, `docs/` packages may be missing from bundle | Add hidden imports for new modules added in Phase 10.4 |

### P1 — Should Address

| # | Issue | Impact | Fix |
|:--|:------|:-------|:----|
| 4 | **Bundled `storage/a3.db`** | Ships stale DB with every release | Seed empty DB at first launch instead of bundling |
| 5 | **No Linux build script** | Manual PyInstaller command needed for Linux | Create `desktop/build.sh` |
| 6 | **No release automation** | Release assets built manually | Consider `desktop/release.sh` to build both platforms |

### P2 — Nice to Have

| # | Issue | Impact | Fix |
|:--|:------|:-------|:----|
| 7 | **No code signing** | Windows SmartScreen warning | Sign with certificate (requires paid cert) |
| 8 | **No auto-update** | Users must manually download new versions | Post-v1.0 feature |
